Ringkasan singkat: Eplant dipakai internal untuk monitoring hasil kebun sawit. Tidak ada iklan, tidak menjual data, tidak memakai lokasi atau kamera. Data akun & operasional (termasuk upah) dikelola di cloud (Singapura) dan hanya diakses sesuai peran. Penghapusan akun melalui admin aplikasi.

1. Pengendali Data & Kontak

PT. Sahabat Agro Group adalah pengendali data untuk Aplikasi Eplant.

  • Email privasi: sahabatagro.it@gmail.com
  • Halaman kebijakan ini akan dipublikasikan pada situs Privacy Policy.

2. Cakupan & Cara Akses

Aplikasi digunakan untuk monitoring hasil kebun sawit internal dan ditujukan khusus karyawan. Pengguna masuk dengan NIP + kata sandi, dan akun hanya dapat dibuat oleh admin. Akses data di dalam aplikasi dibatasi oleh peran/role (mis. mandor, asisten, admin).

3. Data yang Diproses

Kami tidak menayangkan iklan dan tidak menjual data. Eplant memproses data berikut seperlunya:

3.1 Data Akun & Identitas Internal

  • NIP/ID karyawan, nama, jabatan, email/telepon (jika digunakan), kredensial login.
  • Sumber: sistem internal perusahaan; akun dibuat oleh admin.

3.2 Data Operasional/Ketenagakerjaan

  • Data upah/produktivitas dan informasi operasional kebun; ditampilkan sesuai role.

3.3 Berkas/Dokumen yang Dipilih Pengguna (opsional)

  • File yang dipilih secara eksplisit dari perangkat untuk mendukung proses kerja.
  • Aplikasi tidak menggunakan kamera dan tidak mengambil metadata lokasi (geotag) dari file.

3.4 Analitik Internal

  • Metrik penggunaan tingkat agregat diproses di server internal (tanpa SDK pihak ketiga) untuk peningkatan layanan.

Tidak dikumpulkan: lokasi/GPS, audio/mikrofon, foto/video dari kamera, kontak, kalender, pesan, aktivitas di luar aplikasi.

4. Tujuan Pemrosesan

  • Fungsi inti aplikasi: menampilkan/mengelola data operasional sesuai role.
  • Keamanan & audit dasar: autentikasi, pembatasan akses.
  • Analitik internal: pemantauan penggunaan fitur tanpa SDK pihak ketiga.

5. Izin Aplikasi

  • Files/Photos: untuk memilih & mengunggah dokumen yang diperlukan alur kerja.
  • Notifications: belum digunakan. Jika diaktifkan di masa depan, kebijakan ini akan diperbarui.

6. Penyimpanan, Lokasi Server, & Retensi

  • Data disimpan penuh di cloud dengan lokasi server di Singapura.
  • Tidak ada penyimpanan permanen di perangkat (selain cache sementara oleh sistem).
  • Retensi: data operasional & upah disimpan berkelanjutan/selama diperlukan untuk tujuan bisnis & pencatatan, kecuali diminta penghapusan dan tidak bertentangan dengan kewajiban hukum.

7. Berbagi Data

Kami tidak membagikan data kepada pihak ketiga untuk kepentingan mereka sendiri dan tidak menjual data. Jika ada penyedia layanan TI yang membantu pengoperasian sistem, mereka bertindak sebagai pemroses atas nama kami sesuai perjanjian dan standar keamanan.

8. Keamanan

  • Enkripsi saat transit (TLS/HTTPS) dan enkripsi pada penyimpanan server.
  • Penyimpanan kredensial secara aman (mis. hash kata sandi/token).
  • Kontrol akses berbasis peran (RBAC), pemisahan lingkungan staging & production.
  • Kebijakan remote wipe/perangkat hilang diberlakukan secara internal.

9. Hak Pengguna

Pengguna dapat melihat dan memperbaiki data tertentu di aplikasi. Permintaan akses, koreksi, atau penghapusan akun diajukan melalui admin aplikasi internal. Sebagian data dapat dipertahankan/dianonimkan untuk kepatuhan, audit, atau keperluan operasional.

10. Perubahan Kebijakan

Kebijakan ini dapat diperbarui sewaktu-waktu. Perubahan material akan diinformasikan melalui aplikasi atau saluran internal. Tanggal “Terakhir diperbarui” di atas menunjukkan versi terbaru.
